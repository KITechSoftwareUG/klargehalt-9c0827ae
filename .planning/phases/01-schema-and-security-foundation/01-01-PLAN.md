**PART 4: user_roles unique constraint fix**

```sql
-- Remove duplicates keeping newest
DELETE FROM user_roles a USING user_roles b
WHERE a.created_at < b.created_at
  AND a.user_id = b.user_id
  AND a.organization_id = b.organization_id;

-- Drop old wrong constraint
ALTER TABLE user_roles DROP CONSTRAINT IF EXISTS user_roles_user_id_key;

-- Add correct composite constraint
DO $$ BEGIN
  IF NOT EXISTS (SELECT 1 FROM pg_constraint WHERE conname = 'user_roles_user_org_unique') THEN
    ALTER TABLE user_roles ADD CONSTRAINT user_roles_user_org_unique UNIQUE (user_id, organization_id);
  END IF;
END $$;
```

**PART 6: Enable RLS on all tables and create correct policies**

For every tenant-scoped table, `ALTER TABLE ... ENABLE ROW LEVEL SECURITY` then `ALTER TABLE ... FORCE ROW LEVEL SECURITY`.

Standard tenant isolation policy pattern for most tables:
```sql
CREATE POLICY "tenant_isolation" ON {table}
  FOR ALL TO authenticated
  USING (organization_id = public.org_id())
  WITH CHECK (organization_id = public.org_id());
```

Apply this pattern to: `organizations`, `companies`, `departments`, `job_levels`, `job_profiles`, `pay_bands`, `salary_history`, `pay_gap_snapshots`, `compliance_reports`, `profiles`, `info_requests`.
